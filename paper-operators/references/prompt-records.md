# Prompt Records

Use this reference for public examples, README/showcase images, article packages, client work, or any image likely to be reused.

Prompt records make Paper Operators auditable and teachable.

## Minimal Record

Save or report:

```text
asset name:
source anchor:
asset role:
relationship type:
operator family:
truth constraints:
reference cues:
final prompt:
output path:
rejected attempts:
QA result:
caption:
```

## Rejected Attempts

Record a rejected attempt when it teaches a reusable lesson:

- label misspelling
- operator decorative
- wrong data
- wrong historical/scientific cue
- too much in-image text
- crop or edge pressure
- too generic / fails Swap Test

Do not keep noisy notes for every ordinary draft.

## File Placement

For repo examples:

- put final full-resolution images in `examples/images/showcase/`
- put compressed README images in `examples/images/readme/`
- put longer prompt records in `examples/` or a task-specific `PROMPTS.md`

For article packages or external projects, use that project's asset folder.
