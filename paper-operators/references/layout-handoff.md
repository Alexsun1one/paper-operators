# Layout Handoff

Use this reference when a Paper Operators image will be placed inside another artifact: README, article, WeChat post, social card, slide, doc, dashboard, landing page, or knowledge base.

Paper Operators creates the reasoning image. The outer layout may own the headline, body copy, call to action, platform ratio, and caption.

## Handoff Fields

Return these fields when planning a generated asset:

```text
final container:
image role:
recommended ratio:
safe margin:
headline inside image: yes/no
caption outside image:
crop guidance:
background requirement:
handoff owner:
```

## Text Ownership

### Image Owns

Use in-image text for:

- object labels
- route labels
- short state labels
- exact data labels that belong to a chart object
- story beat markers

### Outer Layout Owns

Keep outside the image:

- title
- subtitle
- paragraph explanation
- citations
- long conclusions
- author note
- CTA
- platform-specific metadata

If a generated image already contains too much headline or paragraph text, regenerate with fewer labels instead of relying on layout cropping.

## Ratio Guidance

- README / article figure: 16:9 horizontal.
- Social-card image well: wide crop with strong safe margins; avoid edge labels.
- Slide center image: 16:9 or 4:3 depending on the deck; leave text-safe zones if the slide will add title and bullets.
- Story/book card: 16:9 unless the final platform needs portrait.
- Icon or sticker: usually not Paper Operators.

## Handoff Notes

In final delivery, include:

- full image path
- compressed preview path if created
- recommended caption
- any text that should stay outside the image
- any crop warnings
- whether the image is safe for small display
