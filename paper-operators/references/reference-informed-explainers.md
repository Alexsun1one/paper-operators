# Reference-Informed Explainers

Use this reference when the subject has real-world visual facts: science, history, geography, brands, model families, artifacts, buildings, species, instruments, cultural objects, or technical apparatus.

References are for truth and recognition, not for copying style.

## When To Gather References

Gather references when:

- the user names a specific object, organism, place, brand, model, historical event, scientific mechanism, or cultural artifact
- a wrong silhouette or wrong part would mislead the reader
- the audience expects recognizability
- the topic has changed recently or is niche

Skip reference gathering only when the subject is generic or explicitly symbolic.

## What To Extract

Record:

```text
what it is:
must-have parts:
stable visual cues:
process or mechanism:
common confusions:
source notes:
uncertainty:
```

Translate references into paper-world cues:

- silhouette
- object family
- part layout
- route/map structure
- era/context cue
- symbolic artifact
- color convention only when meaningful

Do not paste logos, trace source diagrams, or mimic another artist's layout.

## Prompt Block

```text
Reference-informed cues: [subject] should include [must-have parts/cues]. Translate these cues into the Paper Operators world with off-white folded paper, charcoal outlines, quiet labels, and one operator performing [action]. Do not copy source images or add unsupported details.
```

## QA

Accept if:

- the subject is recognizable to the intended audience
- key parts are present and labeled correctly
- uncertainty is not hidden as fact
- the operator still performs a relation rather than posing beside a reference model

Reject if:

- a scientific mechanism is wrong
- a historical or cultural scene implies a false event
- a brand/model cue is confused with a different entity
- reference detail overwhelms the Paper Operators argument
