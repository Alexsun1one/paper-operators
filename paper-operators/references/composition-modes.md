# Composition Modes / 构图模式

Use this reference when the content holds several beats and the question is whether to spread them across a series or hold them inside one dense image. The chaining layer in `references/series-and-chaining.md` already supports two moves: intra-image threading (one carrier ribbon, anchor → action → result) and inter-image chaining (a progressing set of figures). This reference adds the decision between those and a third option — a single image that carries MANY beats at once. Choosing the mode the content's own structure demands, instead of defaulting to a series or the flat "three cards on a ribbon," is part of precision. A campaign that crosses the same river four times reads better as ONE map than as four pictures.

Read this with `references/series-and-chaining.md` (when to chain across images), `references/primitives.md` (the kit these layouts are built from), and `references/state-coding.md` (the encoding that keeps many beats readable in one frame).

## Mode Decision: One Dense Image vs a Multi-Image Series

Pick the mode from the content, not from habit. Run these criteria before choosing primitives or planning a set.

| Signal | → One dense image / 单图多拍 | → Series / 串联 |
|--------|------------------------------|-----------------|
| Beats | 2–6 beats on ONE shared stage, terrain, or timeline | genuinely separate worlds, rooms, or time-jumps |
| Shared ground | every beat lives on one map, one system, one body | each beat needs its own world rebuilt |
| Reader value | the takeaway IS seeing all beats together at once | the reader is better paced one beat per scroll |
| Context | a single figure, a pull quote, a header card | a long article needing 2–6 paced body images |

Rule: when the beats share one stage and the reader's takeaway is the simultaneity — four crossings on one terrain, a whole system in one view, before and after in one breath — choose one dense image. When the beats are separate worlds or the article needs paced scrolling, choose a series and defer to `references/series-and-chaining.md`. Do not default to either side; the content's own structure decides.

Multi-beat is not the same as the intra-image chaining already in `references/series-and-chaining.md`. Intra-image chaining threads the phases of ONE relation — anchor → action → result — along a single ribbon, where each stop is a stage of the same idea. Multi-beat holds several DISTINCT events or positions that all belong to one ground: four crossings, three regions, before and after. If the beats are phases of one action, use a plain threaded route; if they are separate happenings on shared terrain, use a multi-beat layout here.

## Single-Image Multi-Beat Layouts / 单图多拍构图

The key new capability. Each layout holds several beats in one frame. Readability comes from the same kit every time: number the beats, encode state with `references/state-coding.md`, give ONE focal beat the focus accent, and protect quiet zones. The layouts are built from the primitives in `references/primitives.md`; only the ARRANGEMENT is new.

### Threaded Route / 一线穿珠
One continuous carrier ribbon (`Ribbon / 绸带`) threads 2–6 numbered beats across a single stage. The operator walks the ribbon and touches it at the focal beat.
- Use for: a sequence of steps that share one stage or tool, where order matters and the whole arc is the point.
- Primitives: `Ribbon / 绸带`, number `Tab / 纸签`, optional `Route Strip / 路径条` underneath.
- Keep readable: number each stop with a small folded number tab; thin the ribbon to quiet gray where it is done and keep it passage blue where it is live; let ONE station carry the focus accent; leave breathing room between stations.

### Looping / Back-and-Forth Route / 往复穿行
A route that crosses the SAME feature several times — the 四渡 pattern. One terrain, one river, one road; the path folds back over it repeatedly.
- Use for: a campaign, maneuver, or argument that revisits the same ground, whose meaning is the REPEATED crossing rather than four separate places.
- Primitives: `Ribbon / 绸带` in the `Looping / 环回` and `Climbing / 攀升` variants; the crossed feature as a `Ribbon` or `Soft Boundary / 软边界`; numbered crossing markers.
- Keep readable: put a numbered marker on the feature at each crossing; vary direction with the carrier-ribbon variants; color the live route as the single focus accent against a neutral terrain; never let two crossings read as equal weight.

### Before → After in One Frame / 前后同框
A split or twin stage holds the starting state and the ending state in one image, joined by a short carrier ribbon or a transformation arrow across the seam.
- Use for: a transformation, repair, or contrast where the punchline is the change itself, seen in one glance.
- Primitives: one shared `Frame / 画框` or a seam `Soft Boundary / 软边界`; a connecting `Ribbon / 绸带` or `Bridge / 桥` across the seam.
- Keep readable: one shared frame or a clean seam; let state coding carry the change (torn → mended, dim → lit, scattered → sorted from `references/state-coding.md`) without extra text; the "after" side holds the focus accent.

### Radial / Clockwise Cycle / 环形循环
Beats arranged around a center and joined by a carrier ribbon that loops back to its start — a `Looping / 环回` ribbon closed into a ring.
- Use for: feedback loops, recurring processes, and anything that returns to where it began.
- Primitives: a `Looping / 环回` ribbon closed to a ring; a center `Plinth / 底座` for the focal beat or a quiet `Mat / 衬垫`; `Route Sign / 路牌` markers around the orbit.
- Keep readable: place 3–5 beats clockwise on one orbit; number them so the entry point is obvious; keep the center as a quiet zone or hold the single focal beat there; a returning arrow tab marks it as a cycle, not a line.

### Nested Zoom / 嵌套缩放
The whole system sits in the frame, and one corner or inset magnifies a detail of it — a `Lens / 镜片` or `Viewing Window / 取景窗` opening onto a blown-up piece.
- Use for: a part-whole story where the reader must see the system AND one detail that explains it.
- Primitives: `Lens / 镜片` or `Viewing Window / 取景窗`; a thin `Thread / 牵线` leader linking inset to whole.
- Keep readable: keep the whole as the main stage and the zoom as a clearly framed inset; connect them with the leader line; the inset holds the focus accent; do not zoom two things at once.

### Terrain / Map Holding a Whole Campaign or System / 地形沙盘
One high-angle paper diorama IS the stage; the beats are routes, positions, or regions placed on it. The terrain is the through-spine and every beat is a mark on the same ground.
- Use for: campaigns, systems, networks, or any content whose beats are all POINTS ON ONE GROUND — exactly the case where four separate images would be wrong.
- Primitives: `Depth layers` for the terrain; `Ribbon / 绸带` routes on its surface; `Plaque / 铭牌` and `Route Sign / 路牌` to name regions and crossings.
- Keep readable: build the terrain from layered paper; hold routes as ribbons on its surface; one route or one breach is the focus accent; the rest stays quiet.

### Cutaway with Layered Beats / 剖切分层
A single object sliced open to show its beats as layers, top-to-bottom or front-to-back — stacked `Slab / 层板` layers inside one body.
- Use for: a system, machine, building, or process whose beats are its internal layers seen together.
- Primitives: a `Slab / 层板` stack inside one body; `Tab / 纸签` on each layer; elevation from `references/state-coding.md`.
- Keep readable: one clean cut; number and name each layer on its tab; use elevation (`Raised` active, `Recessed` done); the focal layer is the one lit or pulled forward.

## Packing Density Without Crowding / 密而不挤

This is why the style is loved: high information, but never crowded, never flat. A single dense image is only better than a series when it stays legible. Density survives crowding through five moves, all already in the kit:

- **Depth layering** — foreground action, mid-stage objects, back shelves (`Depth layers` in `references/primitives.md`). Raise what is active, recess what is done. The 3D paper stage is what lets many beats coexist without colliding; a flat layout cannot hold them.
- **Grouping** — beats that belong together share one tray, shelf, region, or color group; the eye reads clusters, not a wall of equal objects.
- **One focal beat** — exactly one beat carries the focus accent (`Focus Accent · 焦点色`); every other beat stays quieter, in off-white, quiet gray, or a thinner ribbon. Density without a dominant beat is a dashboard.
- **The route as through-spine** — a carrier ribbon threads the beats into one path, so the eye has a single route to follow instead of scanning.
- **Quiet zones** — generous margins and empty paper around the focal beat; dense does not mean full. Protect the quiet zone the way the article protects its key sentence.

Scale beats to the article, not to the canvas. Two or three beats stay generous; five or six need tighter grouping and a stronger single focal beat to avoid a dashboard. The encoding budget from `references/state-coding.md` still applies: pick the 1–2 encodings that carry the precision (numbering + one route color, or elevation + a breach marker) and let everything else stay neutral.

## Worked Example — 四渡赤水 in One Image / 单图案例

Four crossings on one terrain reads as ONE map, not four images. This is the precise case for the looping route.

Composition:

- One high-angle paper terrain diorama — folded mountains and valleys as the stage.
- The 赤水 as a blue paper-ribbon river (`Ribbon / 绸带` in passage blue 流蓝) flowing across the terrain — the fixed feature every crossing returns to.
- The Red-Army route drawn over the river FOUR times as a continuous red focus accent (焦点色, the single thing the eye must follow), folding back across the same water:
  - 一渡西进 / first crossing, heading west;
  - 二渡东返取遵义 / second crossing, return east to take 遵义;
  - 三渡西渡佯动 / third crossing west, a feint (佯动);
  - 四渡东渡南下突围过金沙江 / fourth crossing east, then south to break the encirclement and cross the 金沙江.
- A gray enemy ring (`quiet gray · 静灰`) drawn around the Red Army, with the breach point marked in risk coral 风险珊瑚红 where the ring breaks on the fourth move.
- The commander — one folded-paper operator — at the map edge, pointing along the route, with ONE simple expression: focus through the first three crossings, easing toward relief at the breakout. Calm or a few hand-drawn line strokes, looking at the work, never head-on, never a mascot.

Why one image beats four: all four beats share ONE terrain and the reader's takeaway IS seeing the repeated crossing of the same river — simultaneity is the argument. Four separate images would each show one crossing and lose that the Red Army crossed the same water four times. Numbering (一 / 二 / 三 / 四) plus state coding (red live route, gray ring, coral breach) keep every beat legible in one frame.

Do not draw: four separate maps; a flat 2D map with no paper depth; two crossings sharing the red accent so the route fragments; a busy miniature battle with no readable labels.

Palette check: passage blue river (terrain) + quiet gray ring (enemy) + risk coral breach (danger) + one red focus accent (route) — three semantic colors plus the single focus accent, inside the budget from `references/state-coding.md`.

## When a Series Still Wins / 仍用串联

Choose `references/series-and-chaining.md` instead of a single dense image when:

- The beats are genuinely separate worlds, rooms, or time-jumps with no shared stage — forcing them onto one terrain would distort the content.
- The article is long and needs paced body images the reader scrolls through one beat at a time.
- Each beat deserves its own focal module and quiet zone; cramming them together would make focal beats compete.

When a series wins, still mine `references/variation-engine.md` to vary the frames within the shared world, and keep one throughline ribbon advancing across the set.

## Avoid

- Forcing beats that belong to separate worlds onto one terrain, or splitting beats that share one ground across four images.
- Four equal-weight cells with no focal beat, called "dense."
- A flat timeline or flat map with no depth — a multi-beat layout is still a 3D paper stage, never a paper-textured chart icon.
- Two beats sharing the focus accent so neither can be read.
- A decorative carrier ribbon that does not actually thread the beats.
- Adding beats until the scene becomes a dashboard; if a third encoding is needed, ask whether the wrong mode was chosen.

## Accept / Regenerate — Single Dense Image

### Accept
- Every beat is legible and individually numbered or state-coded in one frame.
- Exactly one beat carries the focus accent; the rest stay quieter.
- Quiet zones survive around the focal beat; the image is dense, not full.
- A carrier ribbon, route, or shared terrain threads the beats into one readable path.
- The scene is a 3D paper stage with depth layers, not a flat chart of equal cells.
- The mode choice matches the content's structure (shared ground → one image; separate worlds → series).

### Regenerate
- The image becomes a crowded dashboard of equal-weight cells with no focal beat.
- It flattens into a paper-textured chart icon or a flat icon grid (a flat timeline, a flat map with no depth, a flat pyramid).
- Beats collide or share the focus accent so none of them can be read.
- Numbering or state coding is missing and the beats cannot be told apart.
- The reader would understand the content better with four separate images — a sign the wrong mode was chosen.
