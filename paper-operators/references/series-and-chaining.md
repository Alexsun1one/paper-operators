# Series and Chaining / 串联

Use this reference when an article needs more than one body image. Most publish-grade articles need 2–6 figures. Readers feel quality when the set reads as one system — one world, one argument, one visual language — not as N unrelated pictures. Chaining means two things at once: continuity across images (inter-image) and a readable throughline within each image (intra-image).

## Two Kinds of Chaining

### Intra-image chaining / 图内串联

One carrier ribbon threads anchor → action → result inside a single image. Multiple relations link into one path: the reader follows a blue ribbon from the source object, through the operator's action station, to the outcome. Labels mark each stop; the operator moves along the ribbon, not beside it.

Accept: one blue carrier ribbon with 2–4 labeled stations; operator touches ribbon, frame, gate, or object at the focal point; start/change/result visible in one scene.

Reject: equal-weight objects with no path; operator standing beside a pretty arrangement; labels scattered without route logic.

### Inter-image chaining / 图间串联

The set shares a world, operator identity, palette, label style, and a progressing throughline. Image 2 advances the same argument — "same article, next beat" — not "new poster."

Accept: same operator family, label container style, high-angle camera, and tabletop world; throughline state progresses image to image.

Reject: image 3 looks like a different style guide; each image repeats the same scene; the set could be reordered without losing meaning.

## Continuity Dimensions

Plan what STAYS constant vs what CHANGES per image.

**Stays constant / 保持不变:** operator identity (same folded-paper body, charcoal outline, folded tile head, semantic color tab); paper material and palette (off-white paper, ink, quiet gray, passage blue carrier; evidence amber, risk coral, growth mint used consistently); label style (one container type: stitched tags, gallery plaques, route signs, or hanging labels); high-angle aerial camera; carrier-ribbon language (smooth blue ribbon, same width/curve); metaphor world (gallery workbench, route table, soft boundary room, archive table, or miniature town — do not switch mid-set).

**Changes per image / 每张变化:** specific relation from `references/relationship-grammar.md`; scene focus (new action station, prop cluster, or state board); operator action (new physical verb); throughline state (loose → taut, scattered → sorted, dim → lit, heavy → placed); labels (new short labels per relation; shared motif labels may return).

## Series Roles

Assign each image a narrative role: opening → build → turn → resolution.

| Role | 角色 | Job |
|------|------|-----|
| Opening / 开场 | Frame question or world | Anchor tension, messy input, unanswered state |
| Build / 展开 | Explain mechanism | Operator routing, framing, sorting, placing |
| Turn / 转折 | Tension or contrast | Risk, inspection, boundary, reframing |
| Resolution / 收束 | Payoff | Delivered result, focused artwork, breathing room |

**Recommended patterns:**

| Length | Pattern | Example rhythm |
|--------|---------|----------------|
| 2 | Opening+Build → Resolution, or Opening → Resolution | 问题 → 结果; 杂乱 → 被看见 |
| 3 | Opening → Build/Turn → Resolution | 开场 → 动作 → 收束 |
| 4 | Opening → Build → Turn → Resolution | 问答 → 派活 → 校验 → 交付 |
| 6 | Opening → Build A → Build B → Turn A → Turn B → Resolution | 问题 → 拆解 → 连接 → 阻力 → 校验 → 收束 |

When the article allows, include at least one human-domain image (art, life, psychology) — do not make every image a product workflow.

## The Throughline Ribbon / 主路径串联

One continuous blue carrier ribbon runs across the whole set. Image N ends where image N+1 begins. The ribbon is the article's argument made visible.

**Handoff rules:** end state of image N = start state of image N+1 (ribbon position, object state, operator posture). State the handoff in the series plan. Operator and ribbon progress together; do not reset to idle unless the article explicitly resets.

**State progression axis** — pick one and advance each image:

| Axis | Start | End |
|------|-------|-----|
| Tension | loose, scattered, dim, heavy | taut, sorted, lit, placed, delivered |
| Attention | noisy field | focused module |
| Boundary | flooded, merged | separated, breathable |
| Evidence | unverified pile | stamped, verified card |

Labels should reflect current state (e.g., 松散 → 拉紧 → 校验 → 交付).

## Motif & Callback / 原元素串联

A recurring small primitive or label returns across the set and pays off in the final image — the "原元素串联" feeling. Pick one prop from `references/primitives.md` (weather card, lamp, stamp, empty frame, evidence dot, gate latch, note tag). Introduce early, transform mid-set, return transformed in resolution. Keep same paper material and label style; change state only.

**Per-image relation vs set motif:** relation comes from `references/relationship-grammar.md` (what one figure explains); motif is the recurring primitive tying figures together. One image = one relation; the whole set = one motif arc.

Example: blue work ribbon (throughline) + verification stamp (motif) un-inked in image 2, stamped on the deliverable in image 4.

Add per-image `role` (opening / build / turn / resolution) when planning. Checklist items: same operator identity; same label container style; same high-angle camera and world; throughline progresses (image N end = image N+1 start); motif introduced and pays off; no redundant images; at least one human-domain image when the article allows.

## Avoid

- Same scene repeated with only label swaps
- Disconnected styles between images
- Redundant images that restate the same relation
- A set that is just one idea × N (no progression, no turn)
- Throughline that does not progress
- Engineering props forced into art/life articles without reason
- Switching operator families without narrative cause

## Accept / Regenerate — Series Coherence

### Accept

- The set reads as one system in under ten seconds of scrolling.
- Shared world, operator identity, palette, and label style are visible across all images.
- Each image has a distinct relation and role; none are redundant.
- The throughline ribbon or state axis progresses; image N end = image N+1 start.
- A recurring motif appears, transforms, and pays off in resolution.
- Intra-image chaining is clear: one carrier path per image, operator on the path.
- The set would weaken if any image were removed or reordered without reason.

### Regenerate

- Images look like different style guides or prompts.
- Throughline does not progress; every image shows the same state.
- The set is one idea repeated N times with cosmetic variation.
- Operator resets to decorative standing; label style jumps between containers.
- Motif is introduced but never pays off; reordering images does not change meaning.
- Cross-image handoff is broken.

### Repair moves

If the set feels disconnected:

```text
Regenerate the series plan first. Lock shared world, operator family, palette, and label style. State the throughline axis and per-image throughline state. Ensure image N end = image N+1 start before regenerating individual images.
```

If the throughline does not progress:

```text
Regenerate with an explicit state axis (loose→taut, scattered→sorted, dim→lit, heavy→placed). Each image must show a different throughline state on the blue carrier ribbon and matching labels.
```

If motif never pays off:

```text
Regenerate the resolution image so the recurring primitive returns in its transformed state. The reader should recognize the motif from image 1 or 2.
```
