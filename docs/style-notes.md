# Style Notes / 风格笔记

## 这次强化的参考图特征

用户明确喜欢的不是某个 IP，而是以下视觉能力：

- 高空视角：能同时容纳路径、空间、标签和多个信息物件。
- 立体元素：每个信息点都像真实纸模、卡片、托盘、门、灯、框、文件盒。
- 流畅线条：主线不是普通箭头，而是有厚度、有走向的蓝色路径带。
- 信息多但不挤：场景丰富，仍有清楚焦点和留白。
- 标签排布漂亮：中文标签像贴纸、展签、吊牌、卡片，贴在对象附近，而不是漂浮在空中。

## 默认生成策略

优先让图像模型画：

- 纸模空间
- 纸片人动作
- 简单图 3-6 个短中文标签；复杂图可以更多，但必须成组、贴对象、读得清
- 蓝色路径
- 立体道具

谨慎让图像模型画：

- 长中文标题
- 多行说明文字
- 客户可见的精确术语
- 法务、金融、医学等高风险文字

默认不要给空白标签。只有明确需要后期叠字、法务/金融/医学高风险文字、客户名称、标题级大字时，才用空白标签容器。现在的默认倾向是让图里直接有可读中文：短标签负责对象和状态，一条短句可以负责最终判断。

## 复杂度判断

复杂度让文章自己决定，不要硬性规定只能简单。先选择最小足够复杂度，但如果文章本身涉及多状态、多主体、多路径、多层关系，可以画复杂图。

复杂图成立的条件：

- 主路径清楚。
- 连线有方向、有分组、有起点和终点。
- 状态有视觉编码，例如颜色、层级、卡片状态、门的开合、灯光强弱、路径粗细。
- 标签足够多且排布漂亮，不让读者猜谜；多标签要按路径、状态、分组或层级组织。
- 复杂元素服务文章结构，不是为了炫技。

简单图成立的条件：

- 一个核心判断足够强。
- 3-6 个短标签，每个标签 2-6 个汉字。
- 纸片人动作一眼可见。
- 留白稳定，读者三秒能知道在讲什么。

## 反向约束

不要滑向：

- PPT 流程图
- stock icon grid
- 小黑式黑色 blob 小人
- robot / AI brain / glowing circuit wallpaper
- 角色很可爱但和文章无关
- 文字满屏但读不清
- 空白标签很多但读者不知道在讲什么
